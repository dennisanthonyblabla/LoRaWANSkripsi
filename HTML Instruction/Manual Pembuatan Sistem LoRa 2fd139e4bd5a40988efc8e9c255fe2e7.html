Manual Pembuatan Sistem LoRa

Hardware yang digunakan :

  • HT-M00 LoRa Gateway
  • Heltec Wifi LoRa V2 Node

Software yang digunakan:

  • The things Network sebagai server
  • Visual Studio sebagai platform IDE
  • Platform IO sebagai plugin Visual Studio untuk pengembangan IoT
  • Framework LMIC Node untuk mengirimkan payload lewat Node

Proses pengembangan yang dilakukan

Registrasi Hardware pada The Things Network (TTN)

Proses yang dilalui untuk melakukan registrasi the things network secara berurut adalah sebagai berikut:

  • Membuat akun TTN
  • Registrasi Node pada TTN. Pastikan perangkat yang digunakan sudah memiliki template pada server TTN sehingga hanya perlu memilih jenis perangkat yang tepat. Jika belum, maka bisa mencari dokumentasi cara registrasi perangkat yang digunakan pada TTN. Untuk Detail dari registrasi pengaturan perangkat yang digunakan dalam dokumen ini adalah sebagai berikut :
    • Brand: Heltec Automation
    • Model: WiFi LoRa (V2) (Class A OTAA)
    • Profile Region: EU_863_870
    • Frequency Plan: Europe 863-870 MHz (SF9 for RX2 - recommended)
    • AppEUI: Klik tombol “Fill with zeros”
    • DevEUI: Klik tombol “Generate”
    • AppKey: Klik tombol “Generate”
    • End Device ID: masukan nama unik apa saja

    Jika sudah selesai, klik “Register end device”

Sedikit catatan. Pada eksperimen ini penulis menggunakan region Europe 863-870. Tetapi dalam melakukan registrasi, region harus disesuaikan dengan ketentuan masing-masing negara agar tidak melanggar ketentuan pemakaian frekuensi radio. Daftar negara dan frekuensi LoRa yang ditentukan dapa dilihat pada website ini: https://www.thethingsnetwork.org/docs/lorawan/frequencies-by-country/

  • Kemudian dilanjutkan dengan melakukan registrasi Gateway pada TTN dengan proses seperti berikut:
    • Gateway ID: masukan nama unik apa saja
    • Gateway EUI: Biasanya diberikan oleh manufaktur alat, tetapi dalam penggunaan gateway heltec, yakni perangkat yang digunakan penulis, EUI dapat diatur sesuai kebutuhan. EUI yang sama tidak dapat digunakan pada 2 akun TTN yang berbeda karena bersifat unik. Disini penulis akan menggunakan ID default yang dapat dilihat pada gambar pengaturan konfigurasi gateway.
    • Gateway name: sebagai alias didalam aplikasi TTN
    • Gateway description: masukan apa saja untuk mendeskripsikan perangkat
    • Gateway server address: sudah terisi sesuai dengan region yang dipilih pada TTN
    • Frequency Plan: gunakan frequency plan yang sama dengan yang digunakan oleh Node
    • Enforce duty cyle: diaktifkan untuk mengikuti regulasi negara mengenai penggunaan jaringan radio tidak terlisensi.

    Jika sudah disesuaikan maka dapat menekan tombol “Create gateway”

Pengaturan Gateway

Gateway perlu terkoneksi pada jaringan internet untuk mengirimkan data dari node kepada gateway. Pada Pengaturan Gateway beberapa hal yang perlu diperhatikan adalah:

  • WiFi SSID: samakan SSID dengan SSID WiFi yang dituju
  • WiFi PASS: masukan password dari WiFi
  • GatewayID: masukan gateway EUI yang sudah terdaftar pada TTN.
  • CH0 Freq: pengaturan dari frekuensi channel yang digunakan. Sesuaikan dengan frekuensi regional yang sudah diatur pada TTN
  • CH1 Freq: karena gateway yang digunakan mendukung dual channel, maka channel kedua akan digunakan bila channel pertama tidak mendapat jangkauan dengan baik.
  • SERVER ADDR: Alamat server yang digunakan. Address tersebut sudah disediakan dari TTN sehingga hanya perlu mengambilnya dari gateway server address yang digunakan pada TTN

Jika pengaturan sudah sesuai, silahkan menekan tombol “Submit”

Pemrograman Node

Pemrograman Node dilakukan pada IDE Visual Studio menggunakan plugin Platform IO untuk mempermudah pengembangan. Setelah melakukan instalasi Visual Studio dan juga Platform IO, dilakukan pengunduhan file project LMIC Node.

Setelah mengunduh file LMIC Node, masuk kedalam file LMIC Node melalui Platform IO, Kemudian buka file platformio. File ini digunakan untuk pemilihan jenis board LoRa yang sedang digunakan, disini interval dan juga monitor speed dapat diatur layaknya menggunakan Arduino IDE.

Kemudian, masih di file yang sama, aturlah frekuensi yang sedang digunakan, disesuaikan dengan konfigurasi TTN. Cara memilih opsi yang tersedia adalah dengan menghilangkan tanda comment.

Lalu, masuk kepada file selanjutnya yang berada di keyfiles > lorawan-keys_example.h, file tersebut perlu diduplikasi dan kemudian di beri nama lorawan-keys.h. Hal ini dilakukan untuk memenuhi dependency yang dibutuhkan oleh project file dari LMIC Node. Masukan keys yang sesuai dengan yang diisi di TTN. Keys yang perlu diisi tergantung dari mode transmisi LoRaWAN yang digunakan, baik OTAA ataupun ABP.

Jika semua pengaturan sudah selesai, maka bisa dilanjutkan dengan mengatur programming dari LoRa node pada file src > boards > LMIC-node.cpp. Pada file ini sudah disediakan tempat untuk pengguna dapat mengatur pemrograman yang dibutuhkan dengan tanda yang bertuliskan “USER CODE BEGIN” dan “USER CODE END”. Pengguna hanya perlu menaruhkan setiap code diantara kedua tanda tersebut sesuai dengan deskripsi yang diberikan pada file tersebut. terdapat beberapa section dengan tanda yang sama untuk memudahkan pengguna. Proses pemrograman sama persis proses yang dilakukan pada Arduino IDE. Jika membutuhkan library tambahan, maka dapat dilakukan pada Platform IO home.

Pengaturan Payload

Untuk pengaturan Payload, yaitu file yang akan ditransmisikan, perlu dilakukan beberapa persiapan payload sebelum bisa dikirimkan melalui LoRa.

Untuk pengumpulan data, pengguna dapat melakukannya pada section “USER CODE BEGIN” yang kedua, sesuai dengan deskripsi yang terdapat pada code.

Jika pengumpulan data sudah diprogramkan, maka pengguna dapat mengubah data yang dikumpulkan dalam bentuk unsigned 16-bit integer. Hal ini dilakukan agar ruang untuk menyimpan data lebih besar.

Jika sudah dikonversikan, maka pengguna dapat memulai pengaturan data yang dikirimkan. Pertama pengguna perlu menentukan berapa payload length yang dibutuhkan. Payload length adalah jumlah payload buffer yang akan dikirimkan. 1 Payload Buffer dapat menyimpan 8 bit, sehingga jika data diubah menjadi 16 bit, maka akan dibutuhkan 2 Payload Buffer yang berisi data uint16_t. Pada payloadBuffer yang pertama, disimpan 8 bit terakhir, kemudian pada payload buffer berikutnya, disimpan 8 bit pertama dari data dan diikuti dengan bit kosong. Contoh persiapan data dapat dilihat di gambar dibawah.

Payload Formatters

Payload Formatters adalah pemrograman untuk melakukan decode terhadap file yang dikirimkan dari LoRa Node. Payload Formatters dapat diisi di TTN sebagai server penerima transmisi. contoh payload formatters dapat diambil dari file LMIC-Node pada bagian payload-formatters > lmic-node-uplink-formatters.js.

Kesimpulan

Bila seluruh proses sudah selesai, maka Node sudah bisa dijalankan. Tanpa pemrograman, LMIC Node sudah mendukung file dummy untuk testing apakah node dan gateway sudah berjalan dengan baik dalam bentuk pengiriman file uplink counter. Bila sudah dapat muncul pada TTN maka artinya sudah berhasil. Contoh data yang dibaca pada TTN seperti gambar dibawah.
